# Проектен доклад: Личен календар (CLI)

## Глава 1. Увод
Настоящият проект реализира конзолно приложение за управление на личен календар. Основната цел е демонстрация на принципите на обектно-ориентираното програмиране в Java (ООП част 1), включително ясно разделяне на отговорностите, устойчив дизайн и коректно валидиране на входните данни. Системата поддържа работа с XML файлове, търсене на свободни интервали и сливане на данни от два календара.

## Глава 2. Преглед на предметната област
Предметната област включва:
- събития (дата, начален час, краен час, име, бележка);
- неработни дни (официални празници или локално зададени почивни дни);
- команден интерфейс за работа от конзола;
- сериализация и десериализация на данни в XML формат.

Типичните потребителски сценарии включват: създаване на събития, редакция на съществуващи записи, намиране на свободни времеви прозорци и обединяване на два източника на календарни данни.

## Глава 3. Проектиране
Архитектурата следва слоест модел:
- `models` - домейн обекти (`Event`, `CalendarContext`);
- `services` - бизнес логика (`CalendarService`);
- `repository` - абстракция и XML реализация за файлов достъп;
- `commands` - реализиран Command Pattern с отделен клас за всяка команда;
- `parser` - ръчен парсър на вход с поддръжка на двойни кавички.

Приложени са принципите:
- SRP: всеки клас има ясно дефинирана отговорност;
- OCP: нова функционалност се добавя чрез нови команди без промяна на съществуващите;
- DIP: `CalendarService` зависи от абстракцията `CalendarRepository`, а не от конкретна реализация.

## Глава 4. Реализация и тестване
Реализацията е изпълнена със стандартната Java библиотека без външни зависимости. XML обработката е ръчна чрез `BufferedReader`, `String.indexOf`, `String.substring` и `PrintWriter`, съгласно ограниченията на заданието.

Тестването включва:
- нормални сценарии: `open`, `book`, `agenda`, `save`;
- отрицателни сценарии: невалидна дата, неизвестна команда, конфликт при припокриване;
- алгоритмични сценарии: `findslot`, `findslotwith`, `busydays`;
- интерактивно сливане: `merge` с избор за запазване или замяна при конфликт.

Скриптовете са приложени в `test_scenarios.txt`.

## Глава 5. Заключение
Проектът покрива целите на курсовата задача по Java ООП част 1. Реализирана е устойчива и разширяема архитектура, която позволява добавяне на нови команди, нови формати за съхранение и допълнителни проверки за валидност. Изпълнени са изискванията за обектно-ориентиран подход, ръчен парсинг и работа изцяло в паметта между командите за запис във файл.
